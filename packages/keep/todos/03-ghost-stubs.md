## Goal

When a project's specs declare `[TYP:ext]` inputs that NO module in the project produces,
`rune sync` generates `bootstrap/stubs.ts` — a keep module with one trivial GET endpoint per
unfulfilled input that mints placeholder values. It mounts like any module (cake at
`/docs/stubs`), is excluded in production, and **evaporates automatically** when a later-synced
module produces the field. Business code never references it.

## Steps

1. **New pure module** `src/rune/domain/business/rune-stubs/mod.ts` (+ `test.ts`):
   - `planStubs(specs: { path: string; text: string }[]): { name: string; tsType: string }[]` —
     `parse()` each spec; collect (a) every `[TYP:ext]` name that some module's ENT input DTO
     consumes unproduced within its own module, (b) the union of ALL modules' ENT output-DTO
     field names (use `dtoFieldNames` — export it from `rune-manifest/mod.ts`). Result = (a)
     minus (b), with each name's `[TYP]` typeName, deduped, sorted. Parse errors in a spec →
     skip that spec (sync already reports them elsewhere).
   - `renderStubsModule(fields): string` — emits `bootstrap/stubs.ts`:
     - header line: `// Generated by rune sync — DO NOT EDIT.` (same as modules.ts; it is the
       delete-guard);
     - imports ONLY `@mrg-keystone/keep` (`Endpoint, EndpointController, endpointModule`) and
       `class-validator` decorators — NO danet/swagger imports (class-validator decorators emit
       `design:type`, which is all the schema builder needs; verified pattern: keep's
       `e2e/checkout/src/checkout/dto/*.ts`);
     - per field: an inline DTO class (`@IsString() memberId!: string;` etc. by tsType) and a
       handler `mint<PascalName>()` decorated
       `@Endpoint({ method: "get", path: "mint-<kebab-name>", output: <Dto>, stub: true, description: "stand-in for the unbuilt producer of <name>" })`
       returning a deterministic value: string → `"stub-<name>-" + counter++` (module-level
       counter so re-runs mint fresh ids), number → counter, boolean → `true`;
     - exports `export const stubsModule = endpointModule("Stubs", [StubsController]);`.
2. **Wire into sync** (`src/rune/entrypoints/sync/mod.ts`, `ensureBootstrap`):
   - Gather project specs: reuse the existing file-collection used by sync/lint plus
     `isProjectSpec` (from `rune-bindings/mod.ts`) to filter; read texts; call `planStubs`.
   - Fields exist → write/refresh `bootstrap/stubs.ts` (skip write when byte-identical, matching
     the modules.ts pattern). No fields → if `bootstrap/stubs.ts` exists AND starts with the
     generated header, delete it; if it exists without the header, leave it and print a note.
   - If `scanSurfaceModules` found a REAL module named `stubs`, skip ghost generation entirely
     and print `module 'stubs' exists — ghost stubs disabled`.
   - `renderAppRegistry`: when the ghost exists, emit a static import of `stubsModule` from
     `@/bootstrap/stubs.ts` and gate it inside the array:
     `...(Deno.env.get("DENO_ENV") === "production" ? [] : [stubsModule])`. Static import only —
     dynamic `import()` deadlocks under rollup-bundled keep apps.
3. **Tests**:
   - `rune-stubs/test.ts`: unfulfilled input → planned; input produced by ANOTHER module's spec →
     not planned; multi-module union; type mapping; deterministic render snapshot via
     `assertStringIncludes` (header, `stub: true`, the gate is NOT here — it's in modules.ts).
   - Sync-level test (follow existing sync tests if present, else a tempdir integration test):
     project with one ext-consuming spec → sync → `bootstrap/stubs.ts` exists and modules.ts
     contains the gated spread; add a second spec producing the field → sync it → stubs.ts gone,
     modules.ts clean.
4. Run `deno test -A src/rune/domain/business/rune-stubs/ src/rune/domain/business/rune-manifest/`
   and any sync tests you added.
